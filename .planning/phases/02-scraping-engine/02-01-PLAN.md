    ```python
    from abc import ABC, abstractmethod
    from dataclasses import dataclass
    from enum import Enum
    from patchright.async_api import Page

    class FailureType(str, Enum):
        NETWORK_ERROR = "NETWORK_ERROR"
        EXTRACTION_ERROR = "EXTRACTION_ERROR"
        BLOCKED = "BLOCKED"

    @dataclass
    class ScrapeData:
        product_name: str
        price_cents: int
        listing_url: str
        retailer_name: str

    class ScrapeError(Exception):
        def __init__(self, failure_type: FailureType, message: str):
            self.failure_type = failure_type
            self.message = message
            super().__init__(f"{failure_type.value}: {message}")

    class BaseExtractor(ABC):
        @property
        @abstractmethod
        def retailer_name(self) -> str: ...

        @property
        @abstractmethod
        def domain_patterns(self) -> list[str]: ...

        @abstractmethod
        async def extract(self, page: Page, url: str) -> ScrapeData: ...

        def _parse_price_to_cents(self, price_str: str) -> int:
            cleaned = price_str.replace("$", "").replace(",", "").strip()
            return int(round(float(cleaned) * 100))
    ```

    **Each extractor follows this pattern:**
    1. Inherits from `BaseExtractor`
    2. Implements `retailer_name` property returning the retailer string
    3. Implements `domain_patterns` property returning list of matching domains
    4. Implements `async extract(self, page: Page, url: str) -> ScrapeData`
    5. Extraction strategy: Try JSON-LD `<script type="application/ld+json">` first (parse for `@type: Product`), fall back to CSS selectors
    6. Uses `self._parse_price_to_cents()` for price conversion
    7. Raises `ScrapeError(FailureType.EXTRACTION_ERROR, "descriptive message")` if extraction fails
    8. Raises `ScrapeError(FailureType.BLOCKED, "descriptive message")` if page shows CAPTCHA or 403 indicators

    **Extractor specifics:**

    **AmazonExtractor** (`backend/app/scrapers/amazon.py`):
    - `retailer_name = "Amazon"`
    - `domain_patterns = ["amazon.com", "www.amazon.com"]`
    - JSON-LD: look for `@type: Product` with `offers.price`
    - CSS fallback: `#productTitle` for name, `.a-price-whole` + `.a-price-fraction` for price (concatenate with ".")
    - BLOCKED detection: check for `captcha` in page content or response status 503

    **BestBuyExtractor** (`backend/app/scrapers/bestbuy.py`):
    - `retailer_name = "Best Buy"`
    - `domain_patterns = ["bestbuy.com", "www.bestbuy.com"]`
    - JSON-LD: standard Product schema
    - CSS fallback: `.sku-title h1` or `[data-testid="heading"]` for name, `[data-testid="customer-price"] span` or `.priceView-customer-price span` for price

    **WalmartExtractor** (`backend/app/scrapers/walmart.py`):
    - `retailer_name = "Walmart"`
    - `domain_patterns = ["walmart.com", "www.walmart.com"]`
    - Primary: Parse `<script id="__NEXT_DATA__">` JSON for product data in `props.pageProps.initialData.data.product`
    - Fallback: JSON-LD, then CSS `[itemprop="name"]` for name, `[itemprop="price"]` or `span[data-testid="price-wrap"]` for price

    **NeweggExtractor** (`backend/app/scrapers/newegg.py`):
    - `retailer_name = "Newegg"`
    - `domain_patterns = ["newegg.com", "www.newegg.com"]`
    - JSON-LD: standard Product schema (Newegg typically has good JSON-LD)
    - CSS fallback: `.product-title` for name, `.price-current` for price (format `$X.XX`)

    **MicrocenterExtractor** (`backend/app/scrapers/microcenter.py`):
    - `retailer_name = "Micro Center"`
    - `domain_patterns = ["microcenter.com", "www.microcenter.com"]`
    - JSON-LD: standard Product schema
    - CSS fallback: `[data-name]` or `h1.product-title` for name, `#pricing span[data-price]` or `.price > span` for price

    **Each extractor must implement `_try_json_ld` helper** (from RESEARCH.md pattern) as a method on BaseExtractor. Move the JSON-LD parsing logic to BaseExtractor so all 5 extractors share it. The method signature: `async def _try_json_ld(self, page: Page) -> ScrapeData | None`.
