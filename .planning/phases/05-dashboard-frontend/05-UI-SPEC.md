# Phase 5 — UI Design Contract

> Visual and interaction contract for the dashboard frontend. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui |
| Preset | Default (zinc neutral, default border radius, CSS variables mode) |
| Component library | Radix UI (via shadcn) |
| Icon library | lucide-react |
| Font | "DM Sans" (body + labels), "Outfit" (headings + display) — loaded via Google Fonts |

**Aesthetic direction:** Utilitarian-refined. Clean data density with generous whitespace, sharp card edges (border-radius from shadcn default), and restrained color usage. The interface should feel like a well-organized control panel — information-dense but never cluttered. Bold typographic hierarchy does the heavy lifting instead of decorative elements.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, inline spacing |
| sm | 8px | Compact element spacing, input padding-y |
| md | 16px | Default element spacing, card internal padding, form field gaps |
| lg | 24px | Section padding, card grid gap |
| xl | 32px | Page horizontal padding (desktop) |
| 2xl | 48px | Header height, major section separation |
| 3xl | 64px | Page top/bottom margin |

Exceptions:
- Touch targets on mobile: minimum 44px hit area for three-dot menu, bell icon, and card action buttons

## Component Interaction Contracts

### Dashboard Card Grid

- **Loading:** Show 6 skeleton cards (same dimensions as real cards) on initial load
- **Empty:** Show empty state (see Copywriting Contract)
- **Card hover:** Subtle shadow elevation increase (`shadow-sm` to `shadow-md` transition, 150ms ease)
- **Threshold-breached card:** 3px left border in emerald-500, plus a small Badge component reading "Below threshold" positioned next to the price
- **"+ New Query" card:** Dashed border, centered plus icon + "Add Query" text, same card dimensions, hover: border becomes solid

### Three-Dot Menu (DropdownMenu)

- Items: Edit, Pause/Resume (dynamic label), Scrape Now, Separator, Delete (text-destructive)
- Click on three-dot icon stops card click propagation (no slide-over open)

### Slide-Over (Sheet)

- Opens from right side, width: 480px on desktop, full-width on mobile
- Close: X button top-right, click outside, Escape key
- Content scroll: independent of background
- Background: dim overlay (black/50 opacity)
- Each listing row: product name, current price (formatted $X.XX), delta arrow + percentage, "Lowest" badge on lowest-price row
- "View history" link: visually present but disabled (muted text, cursor-not-allowed, tooltip "Available in next update")

### CRUD Modal (Dialog)

- Centered, max-width 480px, overlay backdrop
- Form validation: inline error messages below fields, red border on invalid inputs
- URL list: each URL has a text input + remove button (X icon). Minimum 1 URL required.
- "+ Add URL" button below URL list, secondary variant
- Submit button: "Create Query" for new, "Save Changes" for edit
- Loading state on submit: button shows spinner, disabled until response

### Delete Confirmation (AlertDialog)

- Title: "Delete {query name}?"
- Description: "This will permanently remove this watch query, all its retailer URLs, and scrape history. This cannot be undone."
- Actions: "Cancel" (outline) + "Delete" (destructive variant)

### Bell Dropdown (Popover)

- Trigger: bell icon with overlaid unread count badge (red circle, white text, min-width 20px)
- Badge hidden when count is 0
- Panel: max-height 400px, scrollable
- Each alert row: unread dot (blue, 8px), query name (semibold), price + retailer (body), relative timestamp (muted, right-aligned)
- Clicking alert row: marks as read (optimistic UI — remove dot immediately, PATCH in background)
- Footer: "Dismiss All" button (ghost variant) + "View All" link (navigates to /alerts)

### Toast (Sonner)

- Position: bottom-right
- Content: "{query name} — ${price} at {retailer}" with subtext "Below threshold!"
- Auto-dismiss: 5000ms
- Manual close: X button
- Max visible toasts: 3 (stack)

### Alert Log Page (/alerts)

- Page heading: "Alert Log" (Display size)
- "Dismiss All" button: top-right, destructive-outline variant
- Table columns: Status (unread dot), Query Name, Product, Price, Retailer, Time
- Row click: marks alert as read (optimistic)
- Empty state: see Copywriting Contract
- Sort: newest first (server default), no client-side sort in Phase 5

## Price Formatting

All prices arrive from the API as integer cents. Display rules:

| Context | Format | Example |
|---------|--------|---------|
| Card price (large) | "$X.XX" | "$349.99" |
| Card threshold | "Threshold: $X.XX" | "Threshold: $400.00" |
| Slide-over listing price | "$X.XX" | "$349.99" |
| Alert price | "$X.XX" | "$349.99" |
| Delta percentage | "+X.X%" / "-X.X%" / "0.0%" | "-12.5%" |
| Delta arrow | Up arrow for higher, down arrow for lower, dash for unchanged | |

Utility: Create a shared `formatPrice(cents: number): string` helper that returns `$X.XX` format.
